Menu Items

Once installed and running, BSR is administered via the context menu, which can be accessed by right-clicking on the icon in the System Tray. The menu looks like this:



The menu options are explained in the following table:

Option Explanation
Update Web Info Now This will cause BSR immediately to go and update the web stats rather than wait for the next cycle. You might want to do this if you've just rebooted the router and are waiting impatiently for BSR to "realise" that it can now reconnect!
Update Stats Now This will cause the downstream speed and noise ratio to be updated on the system tray icon. In general, the former does not change and the latter changes regularly.
Run Custom Commands Now If there are any custom commands that are configured to be run after a reboot, they can be run immediately. You will not normally have to select this option. Note that when custom commands are running, the automatic reboot feature is temporarily disabled until a successful connection is made to the Internet again. The reason for this is that it is possible to have a custom command that causes a disconnection, such as a change to the threshold SNR. We do not want BSR to reboot the router during this phase as the custom settings will be lost (or a reboot cycle will ensue).
Pause / Re-start BSR This pauses or re-starts BSR. See The Desktop for more information.
Display Current Stats This opens up a web browser with the stats web page that is seen when logging into the router and looking at the router stats.
Display Detailed Stats This displays, in the default text editor, the detailed diagnostics that are often requested, particularly on the Skyuser forums, when investigating router problems.
Display ADSL Username and Password This displays a dialog that shows the ADSL (not web login) user name and password. With these pieces of information, other routers can be used to connect to Sky Broadband. (Though doing so is unsupported and breaks the Sky BB contract.) Whilst the dialog box is open, BSR continues in the background.
Start Telnet Daemon This starts the telnet daemon on the router. By default, telnet is disabled on the Sky routers. Note that there is no authentication to the telnet login, so in secure environments, it is recommended that telnet be disabled.
Kill Telnet Daemon This kills the telnet daemon on the router.
Reboot Router Now Reboot the router immediately. During such a reboot, the automatic reboot feature is temporarily disabled until a successful connection is made to the Internet again. If you select this option, you will get a confirmatory warning before the reboot takes place in case you change your mind.
Enter / Leave Maintenance Mode This puts BSR into "maintenance mode". This is where the whole application runs as usual except that the auto-reboot functionality is disabled. This is a useful mode when there is router maintenance happening but you don't want to put the whole application on pause. For example, you might want to monitor continuously whilst experimenting with some of the router's settings.
NOTE: It is strongly recommended that you do not enter Maintenance Mode if you are in a reboot cycle. (You will receive a warning if you try to do this.) This is because BSR will have no way to know when the cycle is complete and will therefore stay permanently in the middle of a reboot cycle.
Display Log in Log Viewer This displays the current day's log file in the log viewer. Note that the log files can get quite big (depending on the logging level), so keep an eye on them!
View / Edit Config File This opens up the config file in notepad. From here it can be edited and saved. Doing so requires a restart of BSR for the changes to take effect. Details of the config file can be found here: The Config File.
Display Readme This displays the Readme file (README.TXT) in the default text file editor, probably notepad. Whilst the Readme is open, BSR continues in the background.
Help This displays this Help file. Whilst the Help is open, BSR continues in the background.
About... The About screen. Whilst the dialog box is open, BSR continues in the background.
Exit Exit BSR now. Any open threads are killed off and any threads that have to be killed of (rather than waiting for them to close), are logged in the Event Log.

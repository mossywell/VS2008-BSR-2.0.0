Desktop Icon

Once installed, the application will show as a shortcut icon on the desktop:



Double-click on the icon to start it. Any settings saved in the XML file from when BSR was last run, such as custom commands, will automatically be loaded again. The default values, which are used when BSR is run the first time, are as follows:

Router IP Override None
Router Username admin
Router Password sky
URL Timeout 5000
URLs http://www.google.co.uk
http://www.ebay.co.uk
Custom Commands None (but the config file does have sample commands)

NOTE: The System Tray tooltip can only have a maximum of 63 characters and each end of line (including the last one) takes two characters. If the string to be displayed is longer than 63 characters, it is truncated to prevent an exception (AKA a "crash"). So, if the text appears a bit odd, this is what may have happened. The limitation is an operating system one, not an application one.

The following table explains the meaning of the short Result codes shown by BSR.

Code Meaning
OK The response was received successfully.
PE ProtocolError: The response received from the server was complete but indicated a protocol-level error. For example, an HTTP protocol error such as 401 Access Denied would use this status. For example, you may not have permissions to connect to this web site.
NRF NameResolutionFailure: The name resolver service could not resolve the host name. For example, the name of a non-existent web site was specified in the config file.
CF ConnectFailure: The remote service point could not be contacted at the transport level. This would indicate a low-level networking problem.
T Timeout: No response was received during the time-out period for a request. For example, the web site might be down or running very slowly.
SF SendFailure: A complete request could not be sent to the remote server. For example, the web server specified only accepts HTTPS, but HTTP was specified in the config file.
UE UnknownError: A catch-all for all other errors.
? Data is unavailable.

Pausing BSR

There may be times when BSR needs to be stopped. For example, when planned maintenence is being done that will disconnect the router from the computer. Rather than having to unload BSR, it can be paused by double-clicking on the icon in the system tray. Pausing it effectively disables all of the internal timed functions such as checking the router stats. As a result, the auto-reboot functionality is suspended. However, it does not disable the ability to load the config or log files, for example. Double-clicking on the icon in the system tray again restarts it. Note that this causes BSR immediately to check the router stats and Internet connectivity.

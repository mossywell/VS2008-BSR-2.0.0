Logging

Logging is via two locations:
  1. The log file
  2. The Windows Application Event Log
Nearly all logging is via the log file. However, the only circumstance that the Application Event Log is used is when BSR is unable to write to the log file for whatever reason. This causes BSR to close down immediately and the event is logged in the Windows Application Event Log.

The Log File

The log file is where the normal activity of BSR is logged. This includes various types of information, all of which are prefixed by the local computer's date and time. The actual amount of information shown is customisable. See Configuration for more information.

The name of the log file changes every day and is "Mossywell.BSR.<date>.log". It can be found, by default in Windows XP, in "C:\Documents and Settings\<user name>\My Documents\Mossywell\BSR", or in Vista in "C:\Users\<user name>\Documents\Mossywell\BSR". In addition, this location also contains old log files. The number of old log files that are kept is user-definable, but is 10 by default. Older log files are deleted ("pruned") automatically.

The easiest way to open today's log file is via the Context Menu . The six types of information, along with the associated logging levels are:
  1. ERROR (shown with logging level 1 or above) - This is when an event has occurred that you should know about and which requires your immediate attention.
  2. WARNING (shown with logging level 2 or above) - This is when something hasn't worked but which doesn't affect BSR's ability to continue working. The exception to this is when there's a warning stating that "Auto-reboot may not work". This is a warning that the STATS could not be obtained. If the STATS can't be obtained, then no attempt to reboot the router will be made. See Troubleshooting for more information.
  3. INFO (shown with logging level 3 or above) - This is when the router has done something that you might be interested in such as doing an automatic reboot, or completing a set of custom commands.
  4. COMMAND (shown with logging level 4 or above) - This is when a command has been sent to the router.
  5. STATS (shown with logging level 5 or above) - This contains the following fields separated by a semi colon:
    • The uptime in hours:minutes:seconds
    • The downstream sync speed
    • The downstream noice margin (SNR)
    • The number of received bytes on the PPP interface (i.e. WAN) since last reboot
    • The number of transmitted bytes on the PPP interface (i.e. WAN) since last reboot
    The Internet is not contacted to obtain these stats.
  6. WEBPAGE (shown with logging level 6 or above) - This is when BSR connects to the web sites to check that the Internet connection is up. If there are any errors, the long description is provided in this long file. This long description is the equivalent of the abbreviated description in the tooltip. See Desktop for more information on the short description.
A logging level of 6 (or above) will show "everything" in the log file, and a logging level of 0 will log nothing.
